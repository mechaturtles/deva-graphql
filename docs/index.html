Why Deva?

Deva began as an idea to save developer time and reduce vendor costs by auto-generating a unified GraphQL interface that joins data across business domains without the overhead of a heavy framework.

The goal is to:

  1. Use metadata to drive the super-resolver
  2. Auto-generate new cross-entity fields that join indices
  3. Remain portable enough to run entirely in-browser

While it's designed with Elasticsearch and Lucene in mind, I hope to support any centralized NoSQL store where on-demand joins are difficult to resolve automatically.

MusicBrainz Integration

I'm using MusicBrainz's Lucene search API to showcase Deva's potential. Eventually, this will demonstrate a general-purpose, NoSQL-style interface to any Lucene-based search engine.

Try the test page to explore the MusicBrainz API directly and see how it will integrate with Deva.
